What are FPS?

FPS means frames per second: how many complete images the browser can present every second.
24fps cinema cadence
30fps feels acceptable, not snappy
60fps baseline for fluid UI
120fps same rules, half budget

A 60Hz screen gives 16.67ms

A 60Hz display refreshes 60 times per second, so one frame slot is 1000 / 60 = 16.67ms. Miss that slot and the browser shows the previous frame again. On 120Hz, the budget drops to 8.33ms.

Before rendering pixels, event loop gets a turn

The browser event loop runs a task, flushes microtasks, then gives rendering a chance. If your code overstays, the next frame waits.
Task queue input, timers, event handlers
Microtasks promises, MutationObserver
requestAnimationFrame last safe JavaScript before render
Render work style, layout, paint, composition

What does 60fps mean?

It means delivering 60 fresh frames every second. When you miss one slot, browser repeats previous frame and motion instantly feels late.
