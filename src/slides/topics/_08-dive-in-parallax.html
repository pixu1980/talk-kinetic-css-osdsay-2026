The dive-in component stacks text layers with depth. CSS uses --index, --mouse-x, and --mouse-y custom properties to create parallax, blur, and opacity gradients, all without a single animation in JS. 
      
stack-container {
  --mouse-x: 0;
  --mouse-y: 0;
  ...
}

stack-item {
  --threshold: 0.1 * var(--index);

  position: absolute;
  transform: translate(-50%, -50%);
  filter: blur(100%);
  opacity: calc(1 - (0.2 * var(--index)));
  will-change: font-size, filter, text-shadow;
}

stack-container:hover {
  width: 117rem;
  height: 50rem;

  stack-item {
    top:  calc(50% + (var(--mouse-y)
                      * var(--threshold)));
    left: calc(50% + (var(--mouse-x)
                      * var(--threshold)));
    font-size: calc(15rem
                    - (1rem * var(--index)));
    filter: blur(calc(0.1rem * var(--index)));
  }
}
  • --index is set per item by JS (reversed order: back layers get higher index)
  • --threshold multiplies --index to create depth-dependent parallax displacement
  • On hover the container expands and items unblur, revealing the stacked parallax
  • Opacity decreases with --index: front layer is fully opaque, back layers fade
  • font-size shrinks with --index to add perspective to the stack
The DiveInElement custom element does two things: assigns reversed --index values and writes mouse offset as CSS variables. All visual logic stays in CSS. 
      
class DiveInElement extends HTMLElement {
  static { customElements.define('dive-in',
           DiveInElement); }

  connectedCallback() {
    const items = [...this.querySelectorAll(
                    'stack-item')].reverse();
    items.forEach((item, i) =>
      item.style.setProperty('--index', i));
    document.addEventListener('mousemove',
                              this.handleMouseMove);
  }

  handleMouseMove(event) {
    const container =
      this.querySelector('stack-container');
    const offsetX =
      window.innerWidth / 2 - event.clientX;
    const offsetY =
      window.innerHeight / 2 - event.clientY;
    container.style.setProperty('--mouse-x',
                                `${offsetX}px`);
    container.style.setProperty('--mouse-y',
                                `${offsetY}px`);
  }
}
  • Items are reversed so the last DOM element (visually on top) gets --index: 0
  • Mouse offset is computed from viewport center, creating natural parallax direction
  • JS only writes three CSS custom properties; CSS does all positioning and effects
  • The custom element self-registers via a static initializer block
  • Cleanup happens in disconnectedCallback (event listener removal)

Custom Elements

The Custom Elements API lets you define new HTML elements with their own behavior. Here a custom element is used purely as a CSS variable bridge: it reads user input and writes custom properties that CSS consumes for parallax and depth effects.
